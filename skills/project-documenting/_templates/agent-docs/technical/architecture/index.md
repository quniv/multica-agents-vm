# Architecture

Track system architecture documentation and its status.

## Entry format

Store each architecture page at `arch-NNN_<topic-name>/index.md`, such as
`arch-001_service-boundaries/index.md`.

- Code: The next unused three-digit code, starting with `arch-001`.
- Description: A concise summary of the design or decision.
- Status: `in progress`, `completed`, or `on hold`. Use `in progress` for a new
  page unless another status is specified.

Add the page to this index when creating it. Keep its code, description, and
status synchronized with the architecture page.
