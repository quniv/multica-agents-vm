# Features

Track feature requests and their status.

## Entry format

Store each feature at `feat-NNN_<feature-name>/index.md`, such as
`feat-001_user-authentication/index.md`.

- Code: The next unused three-digit code, starting with `feat-001`.
- Description: A concise summary of the feature.
- Status: `in progress`, `completed`, or `on hold`. Use `in progress` for a new
  feature unless another status is specified.

Add the feature to this index when creating it. Keep its code, description, and
status synchronized with the feature page.
