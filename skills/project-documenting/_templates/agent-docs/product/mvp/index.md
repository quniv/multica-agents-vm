# MVPs

Track minimum viable products and their status.

## Entry format

Store each MVP at `mvp-NNN_<mvp-name>/index.md`, such as
`mvp-001_customer-portal/index.md`.

- Code: The next unused three-digit code, starting with `mvp-001`.
- Description: A concise summary of the MVP.
- Status: `in progress`, `completed`, or `on hold`. Use `in progress` for a new
  MVP unless another status is specified.

Add the MVP to this index when creating it. Keep its code, description, and
status synchronized with the MVP page.
